Wenn du Vorhersagedaten konfiguriert hast, wird dir die geschätzte Restproduktion für den aktuellen Tag im UI angezeigt.
Zudem bekommst du eine Diagramm-Ansicht mit den Daten der nächsten Tage.

Es ist auch möglich, **mehrere PV-Vorhersagen** zu kombinieren, um mehrere Dachflächen abzudecken.
Die Werte werden dafür aufsummiert.
Verwende dafür die folgende Syntax:

```yaml
tariffs:
  solar:
    - type: template
      template: ...
      # ...
    - type: template
      template: ...
      # ...
```

Viele Anbieter haben ein Ratelimit.
Daher musst du ggf. das Update-Intervall (`interval`) erhöhen.

### Anpassung der Vorhersage

Neben der Vorhersage protokolliert evcc auch die aktuelle PV-Leistung und akkumuliert diese über den Lauf der Zeit.
Das passiert auch für die Vorhersagedaten.
Durch den Vergleich der beiden Werte über die Zeit kann evcc die Vorhersagen an die Werte deiner tatsächlichen Produktion anpassen.
Du kannst diese Anpassung im Vorhersage-Popup im UI ein- und ausschalten.

Solltest du Probleme mit den Werten dieser Anpassung haben, kannst du die bereits erfassten Daten zurücksetzen.

Vorgehensweise:

Evcc stoppen

dann über die CLI:
```bash
# Prognostizierte Energie und real erzeugte Energie zurücksetzen
evcc settings set solarAccForecast 0
evcc settings set solarAccYield "{}"
```
Du wirst jeweils gefragt, ob du das wirklich möchtest und kannst dies mit `y` bestätigen.

Danach evcc wieder starten.

Jetzt beginnt evcc wieder von 0 an die Vorhersagen zu akkumulieren.
