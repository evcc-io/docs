# `chargers`

Zur Steuerung der Ladung muss evcc mit einer Wallbox kommunizieren können.

Eine Wallbox hat mindestens folgende Konfiguration:

```yaml
chargers:
  - name: wallbox1 # reference name
    type: ...
    ...
```

Im folgenden werden nun alle möglichen Parameter erklärt.

---

## Erforderliche Parameter

### `name`

Eine Kurzbezeichnung der hier definierten Wallbox. Der Wert wird in der Refernzierung der Wallbox in der Konfiguration des [Ladepunktes](loadpoints#charger) verwendet.

**Beispiel**:

```yaml
  name: wallbox1
```

---

### `type`

Dies ist der evcc spezifische Wallbox Typ, mit Hilfe dessen mit der Wallbox kommuniziert werden kann. Den passenden Typ findet man unter [Geräte - Wallboxen](/docs/devices/chargers).

**Beispiel**:

```yaml
  type: wallbe
```

## Unterstützte Typen

Im folgenden sind die verschiedenen möglichen Typen dokumentiert:

### `abl`

<SponsorshipRequired />

ABL eMH1, erfordert einen Modbus RS485 Adapter

**Beispiel**:

```yaml
  type: abl
  ...
```

---

### `cfos`

<SponsorshipRequired />

cFos PowerBrain charger. [`charge meter`](meters) muss separat konfiguriert werden

**Beispiel**:

```yaml
  type: cfos
  ...
```

---

### `easee`

<SponsorshipRequired />

Easee Home Wallbox.

**Beispiel**:

```yaml
  type: easee
  ...
```

---

### `eebus`

:::warning Experimentelle Unterstützung
:::

Wallboxen die kompatibel zum EEBUS Standard sind.

**Beispiel**:

```yaml
  type: eebus
  ...
```

---

### `evsewifi`

Wallboxen mit dem SimpleEVSE Controller die [EVSE-WIFI](https://www.evse-wifi.de/) verwenden, z.B. smartWB.

**Beispiel**:

```yaml
  type: evsewifi
  ...
```

---

### `go-e`

go-eCharger Wallboxen. Sowohl lokal als auch über die Cloud API.

:::note
Firmware 040.0 oder neuer benötigt!
:::

**Beispiel**:

```yaml
  type: cfos
  ...
```

---

### `heidelberg`

<SponsorshipRequired />

Heidelberg Energy Control

**Beispiel**:

```yaml
  type: heidelberg
  ...
```

---

### `keba`

KEBA KeContact P20/P30 und BMW Wallboxen

**Beispiel**:

```yaml
  type: keba
  ...
```

---

### `mcc`

Mobile Charger Connect von Audi, Bentley und Porsche

:::note
Die Ladung kann hiermit NICHT unterbrochen werden,der PV Modus funktioniert hiermit nicht!
:::

**Beispiel**:

```yaml
  type: mcc
  ...
```

---

### `nrgkick-bluetooth`

NRGkick Wallbox mit einer Bluetooth Verbindung

:::note
evcc muss unter Linux laufen, Docker wird nicht unterstützt!
:::

**Beispiel**:

```yaml
  type: cfos
  ...
```

---

### `nrgkick-connect`

NRGkick Wallbox mit zusätzlichem NRGkick Connect Modul.

**Beispiel**:

```yaml
  type: nrgkick-connect
  ...
```

---

### `openwb`

openWB Wallbox, gesteuert über openWBs MQTT Schnittstelle. Der Parameter `phases: true` muss angegeben werden, falls die Wallbox über eine 1p3p Umschaltung verfügt.

**Beispiel**:

```yaml
  type: cfos
  ...
```

---

### `phoenix-em-eth`

Wallboxen mit dem Phoenix EM-CP-PP-ETH Controller.

**Beispiel**:

```yaml
  type: phoenix-em-eth
  ...
```

---

### `phoenix-ev-eth`

Wallboxen mit dem Phoenix EV-CC-**-ETH Controller.

**Beispiel**:

```yaml
  type: phoenix-ev-eth
  ...
```

---

### `phoenix-ev-ser`

Wallboxen mit dem Phoenix EV-CC-***-SER seriellen Controller (ModBus RTU)

**Beispiel**:

```yaml
  type: phoenix-ev-ser
  ...
```

---

### `simpleevse`

Wallboxen mit dem SimpleEVSE Controller und einer ModBus Verbindung (z.B. OpenWB, Easy Wallbox B163, ...)

**Beispiel**:

```yaml
  type: simpleevse
  ...
```

---

### `wallbe`

Wallbe Eco Wallbox. Für ältere Wallbe Wallboxen (vor 2019) mit Phoenix EV-CC-AC1-M3-CBC-RCM-ETH Controller muss `legacy: true` als Parameter angegeben werden.

**Beispiel**:

```yaml
  type: wallbe
  ...
```

---

### `warp`

Tinkerforge Warp/Warp Pro

**Beispiel**:

```yaml
  type: warp
  ...
```

---

### `custom`

Standard Implementierung, kann über [Plugins](/docs/reference/plugins) für beliebige Wallboxen genutzt werden.

**Beispiel**:

```yaml
  type: custom
  ...
```

---

### `fritzdect`

Fritz!DECT 200/210 Adapter.

**Beispiel**:

```yaml
  type: fritzdect
  ...
```

---

### `shelly`

Shelly adapter.

**Beispiel**:

```yaml
  type: shelly
  ...
```

---

### `tasmota`

Tasmota adapter.

**Beispiel**:

```yaml
  type: tasmota
  ...
```

---

### `tplink`

TP-Link HSXXX Adapter.

**Beispiel**:

```yaml
  type: tplink
  ...
```
