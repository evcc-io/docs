## Template Parameter `homematic`

| Name | Explanation | Value | Optional |
|-----------|-------------|-------|----------|
| IP address or hostname<br/>`host` |  | Beispiel: `192.0.2.2` | yes |
| XML-RPC server port number<br/>`port` |  | Standard: `2010`<br/>Beispiel: `BidCos-Wired=2000, BidCos-RF=2001, HmIP=2010` | no |
| Device address/Serial number<br/>`device` | Homematic device id like shown in the CCU web user interface. | Beispiel: `0001EE89AAD848` | yes |
| Username<br/>`user` | e.g. email address, user id, etc. |  | no |
| Password<br/>`password` | Use single quotes in case of leading zeros |  | no |
| Standby power<br/>`standbypower` | Power values above this value will be considered as charging power | Standard: `15`<br/>Einheit: W | no |
| Meter channel number<br/>`meterchannel` | Homematic meter channel number like shown in the CCU web user interface. | Standard: `6`<br/>Beispiel: `HMIP-PSM=6, HMIP-FSM+HMIP-FSM16=5, HM=2` | yes |
| Switch/Actor channel number<br/>`switchchannel` | Homematic switch actor channel number like shown in the CCU web user interface. | Standard: `3`<br/>Beispiel: `HMIP-PSM=3, HMIP-FSM+HMIP-FSM16=2, HM=1` | yes |
| XML-RPC API cache duration<br/>`cache` | In case of duty cycle problems try a cache setting of 30s. | Standard: `1s`<br/>Beispiel: `5m` | no |
| Integrated device<br/>`integrateddevice` | Integrated device. No charging sessions |  | no |
| Heating device<br/>`heating` | Shows °C instead of % |  | no |
| Icon<br/>`icon` | Will be displayed in the user interface | Auswahl: `car`, `bike`, `bus`, `moped`, `motorcycle`<br/>, `rocket`, `scooter`, `taxi`, `tractor`, `rickshaw`<br/>, `shuttle`, `van`, `airpurifier`, `battery`, `bulb`<br/>, `climate`, `coffeemaker`, `compute`, `cooking`, `cooler`<br/>, `desktop`, `device`, `dishwasher`, `dryer`, `floorlamp`<br/>, `generic`, `heater`, `heatexchange`, `heatpump`, `kettle`<br/>, `laundry`, `laundry2`, `machine`, `meter`, `microwave`<br/>, `pump`, `smartconsumer`, `tool`, `waterheater` | no |
